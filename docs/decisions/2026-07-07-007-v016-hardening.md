# 007. v0.1.6 hardening — injection, privacy, signing, supply chain

**Date:** 2026-07-07
**Status:** Accepted
**Deciders:** Claude (CTO, recommends) + User (CEO, approves)

## Context
The v0.1.5 deep review (kept local, gitignored: `docs/audits/2026-07-07-deep-review-v0.1.5.md`)
found one constitution violation and several majors. This ADR records the decisions taken to
fix them in v0.1.6. The CEO directed "fix everything."

## Decisions
- **Text delivery (B1).** The injection path no longer auto-copies the transcript to
  `NSPasteboard` when Accessibility is missing — Universal Clipboard can carry pasteboard
  contents off-machine (CLAUDE.md privacy rule / profile §35). Instead: 🔐 icon + a
  notification, and a **"Copy Last Dictation"** menu item that copies only on an explicit user
  action. The transcript is held in memory (`lastTranscript`) for that recovery.
- **Injection reliability (M1).** The transcript is posted in **≤16-UTF-16-unit chunks** (never
  splitting a surrogate pair) with a ~1.2 ms inter-chunk delay, **off the main thread**.
  Single-event posting truncated/garbled long text in some apps (the "types worse than paste"
  report). Chunk size is empirical — needs a live A/B on Slack/Chrome/Mail.
- **Secure fields (M2).** Refuse injection when `IsSecureEventInputEnabled()` **or** the focused
  element's AX subrole is `kAXSecureTextFieldSubrole`. **Residual limitation:** some Electron
  password fields expose neither signal; we deliberately do **not** fail-closed on an unknown
  subrole, because that would break typing into ordinary web/Electron fields (the app's whole
  purpose). Documented residual risk.
- **Signing (M4/M5).** The keychain unlock password is now **random, stored only in
  `~/.wisperlocal-signing-pw` (gitignored)**; the previously hardcoded public value was
  **rotated** (now useless) and the key is imported without `-A`. `make-app.sh` **refuses to
  ship an ad-hoc/unsigned release** (`--dev` for local builds) — a silent downgrade would reset
  users' TCC grants.
- **Supply chain (M3/M6).** `install-prebuilt.sh` pins the app to a specific release version and
  **verifies SHA256** of both the app zip and the model against committed constants;
  `make-app.sh` runs a binary **egress scan** (`nm`/`otool`) and fails on any networking symbol.
- **Logging (M7).** A transcript-free file logger (`WisperCore/Log.swift`) writes lifecycle +
  errors to `~/Library/Logs/WisperLocal/`. Its message parameter is `StaticString`, so a
  transcript/audio value is *type-level* impossible to log.
- **Tests (M8).** Added `ModelStore` (RAM tier + path preference), `AudioFile.resampleMono`, and
  an `AudioCapture` leak test — 25 total, all green.

## Deferred (with reasons)
- **m6 (Ctrl+C double-tap false trigger).** The only reliable fix is a global keyDown monitor —
  keylogger-grade surveillance our own profile §coverage avoids. The right fix is the Phase-5
  **rebindable hotkey**; deferred to it.
- **M9 (external review pass).** CLAUDE.md's Codex passes could not run — macOS XProtect
  quarantined the `codex` CLI mid-review. v0.1.5/v0.1.6 were reviewed with Claude
  Quality-Reviewer + adversarial passes only. CEO accepted this deviation; a true external pass
  is optional follow-up.

## Consequences
- Prose dictation into a blocked (no-Accessibility) field now requires an explicit copy.
- The pinned-version installer must have its `APP_VERSION` + `APP_SHA256` bumped every release
  (documented in `docs/DEVELOPMENT.md`).
- Full findings + `file:line` citations live in the local audit doc; this ADR is the shareable record.
